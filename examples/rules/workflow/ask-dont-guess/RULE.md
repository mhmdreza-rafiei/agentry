# ask-dont-guess

Before writing code, changing architecture, or stating product facts, ask when anything is unclear, ambiguous, missing, or a "probably fine" default that would lock the project in.

- Use short, concrete questions; multiple-choice is fine.
- Do not invent missing specs.
- When verified project context already answers it, proceed — do not re-ask settled facts.
- When docs and code conflict, trust verified code and runtime evidence.
